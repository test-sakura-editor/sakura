2 // printf系ラップ関数群
3 // 2007.09.20 kobake 作成。
4 //
5 // 重要な特徴として、独自のフィールド "%ts" および "%tc" を認識して処理する、という点があります。
6 // UNICODEビルドでは "%ts", "%tc" はそれぞれ "%ls", %lc" として認識され、
7 // ANSIビルドでは "%ts", "%tc" はそれぞれ "%hs", %hc" として認識されます。
8 //
9 // "%s", "%c" は使用関数により型が変わり、char, wchar_t が混在するコーディング環境ではバグの元となりやすいので、
10 // できるだけ、上に記したような明示的な型指定をしたフィールドを用いてください。
11 //
12 // 注意：%10ts %.12ts のようなものは未サポート
13 //
14 // ++ ++ 改善案 ++ ++
15 //
16 // あくまでも標準ライブラリ動作を「ラップ」しているだけなので、
17 // そのラップ処理分、パフォーマンスは悪いです。
18 // 標準ライブラリに頼らずに全て自前で実装すれば、標準ライブラリ並みのパフォーマンスが得られるはずです。
19 //
20 // ちょっと関数名が分かりにくいので、もっと良い名前募集。
21 // 今のままだと、上記説明を読まなければ、_tsprintf とかと何が違うの？と思われちゃいそう。。。
22 //
23 // プロジェクト全体がTCHARに頼らないのであれば、これらの関数群は不要。
24 //
